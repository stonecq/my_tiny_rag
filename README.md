# 小型 RAG 项目（基于中文维基百科）

本项目是一个用于学习和实验的**小型检索增强生成**（RAG）系统，使用中文维基百科（wiki_baike）作为知识库，结合开源模型实现端到端的问答流程。

---

## 🧠 核心组件

- **重排序模型**（Reranker）  
  `BAAI/bge-reranker-base`

- **嵌入模型**（Embedding）  
  `BAAI/bge-base-zh-v1.5`

- **大语言模型**（LLM）  
  `Qwen2.5-7B-Instruct`（通义千问系列）

- **文本切分策略**  
  基于规则的句子/段落划分（如标点、长度限制等）

- **数据集**  
  中文维基百科（`wiki_baike`），已预处理为适合检索的格式

---

## 📦 功能特点

- 支持中文问答
- 端到端 RAG 流程：查询 → 检索 → 重排序 → 生成答案
- 模块化设计，便于替换模型或调整切分逻辑
- 适用于本地部署与学习研究

## 📚 学习过程与开发路线

本项目在开发过程中遵循以下学习路径，逐步构建完整的 RAG 系统：

1. **环境搭建与模型下载**  
   配置 Python 环境，下载所需 Hugging Face 模型（embedding、reranker、LLM）

3. **召回模块实现**  
   - 实现 **BM25 稀疏召回**（基于 `rank_bm25`）

4. **数据库构建**  
   对 `wiki_baike` 数据集进行清洗、切分，并构建混合检索索引

6. **RAG 系统集成**  
   融合召回、重排序与 LLM 生成，形成端到端问答 pipeline

## 🔍 检索机制说明

本系统采用 **双路召回 + 重排序 + 上下文融合** 的多阶段检索策略，兼顾关键词匹配与语义相似性，提升答案相关性与连贯性。

### 1. **双路召回**（Hybrid Retrieval）

- **BM25 召回**：基于词频的稀疏检索，擅长匹配关键词、实体名等显式信息。
- **向量召回**：使用 `bge-base-zh-v1.5` 生成稠密向量，通过 FAISS 进行近似最近邻搜索，捕捉语义相似性。

> 默认每路召回 `2 × top_n` 个候选块，确保后续重排序有足够多样性。

### 2. **去重与合并**

将 BM25 与向量召回的结果按 `chunk_id` 去重，合并为一个无重复的候选列表，避免同一文本块被多次处理。

### 3. **重排序**（Reranking）

使用 `BAAI/bge-reranker-base` 对合并后的候选块进行精细化打分，输出最终 **Top-K** 相关片段。  
重排序模型以 `(query, passage)` 对为输入，输出相关性分数，显著优于单纯向量相似度。

### 4. **上下文感知检索**（`search_with_context`）

为提升生成答案的连贯性，系统支持 **上下文合并模式**：
- 每个文本块（`SplitChunk`）记录其原始文档中的上下文位置（`context_ids = [start_id, end_id]`）。
- 在最终结果中，**相邻或重叠的文本块会被自动合并**为连续段落。
- 合并后保留最高相关性分数，并拼接完整上下文（首块含标题/元信息，其余仅正文）。

> 此模式特别适用于需要长上下文支持的问题（如“请总结李白的生平”），避免 LLM 因信息碎片化而生成不完整答案。
